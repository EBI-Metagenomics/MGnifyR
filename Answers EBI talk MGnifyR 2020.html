Boxplots:¶

The following snippet shows one way to build the boxplot. First the diversity and metadata data.frames are joined using the merge command. Since both data.frames have rownames set to accession id, the by=0 parameter is used. Also, all.y is specified so that any "y" entries (in this case analyses_metadata) are dropped if there isn't a corresponding "x" row (div_rare). This makes it a Left Outer join in SQL parlance.

Having generated the combined data.frame, ggplot2 is used to actually build the plot.

Ordination¶

Generating the ordination plot is pretty easy as well, once you know what you're doing. metaMDS from vegan expects a community matrix with samples as rows and taxa as columns, so a transpose (t) is required to flip the OTU table.

merge is again used to join the ordination results with the sample metadata, only this time for illustrative purposes we subset the metadata prior to merging, and specify "row.names" and "analysis_accession" instead of by=0. Either (and more) ways are perfectly legitimate ways to do this.

Finally, ggplot2 is used (again) to plot the points, coloured by the biome type.
